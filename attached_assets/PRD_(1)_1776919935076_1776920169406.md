# Product Requirements Document (PRD)
## Securify: Multi-Agent LLM System for Digital Scam Protection

**Version:** 1.0  
**Date:** April 23, 2026  
**Author:** Diego Carpintero  
**Status:** Active

---

## 1. Executive Summary

Securify is an AI-powered scam detection system that allows users to upload a screenshot of any digital communication (SMS, email, social media post, etc.) and receive an instant, plain-language verdict on whether it is a scam. The system coordinates a pipeline of seven specialized LLM agents, each responsible for a distinct step of the analysis, and returns results in the user's native language.

---

## 2. Problem Statement

Digital scams are a growing global threat. Fraudulent SMS messages, phishing emails, and fake social media promotions cause billions of dollars in financial losses each year and disproportionately affect non-technical users and non-English speakers. Existing solutions are:

- Too technical for average users to operate confidently
- English-only, leaving multilingual users underserved
- Reactive rather than proactive (e.g., spam filters that only block known senders)
- Unable to explain *why* something is suspicious in user-friendly terms

Securify addresses these gaps by providing a multimodal, multilingual, explainable scam detection tool accessible to anyone.

## 5. Scope

### 5.1 In Scope
- Screenshot upload and visual OCR-based text extraction
- URL extraction and safety verification via Google Safe Browsing API
- Scam pattern analysis (urgency, threats, impersonation, social engineering)
- Final scam/not-scam determination with a confidence score (1–5 scale)
- Plain-language summary with actionable recommendations
- Automatic translation of the summary to the source language of the message
- Persistent archiving of all results to a local SQLite database
- Duplicate detection (via SHA-256 hashing) to avoid redundant storage
- Live streaming of the agent workflow in the UI for transparency

### 5.2 Out of Scope
- Real-time monitoring of incoming messages
- Browser extension or mobile app integration
- Direct email/SMS inbox connectivity
- User authentication or account management
- Admin dashboard for database exploration (future phase)

## 6. System Architecture

### 6.1 Technology Stack

| Component | Technology |
|---|---|
| Agent Framework | AutoGen v0.4.0 (autogen-agentchat, autogen-ext) |
| LLM | OpenAI GPT-4o-mini (vision + reasoning) |
| UI | Gradio |
| URL Safety | Google Safe Browsing API v4 |
| Database | SQLite |
| Language | Python 3.12 |

### 6.2 Agent Pipeline (Round Robin)

The seven agents execute sequentially in a fixed Round Robin order:

```
[Image Upload]
      ↓
1. OCR Specialist         → Extracts and cleans text from the image
      ↓
2. Link Checker           → Identifies URLs and checks them against Google Safe Browsing
      ↓
3. Content Analyst        → Detects scam patterns, urgency, threats, inconsistencies
      ↓
4. Decision Maker         → Issues final scam/not-scam verdict with confidence score (1–5)
      ↓
5. Summary Specialist     → Condenses findings into one user-friendly paragraph
      ↓
6. Language Translation   → Translates summary to the source language of the message
      ↓
7. Data Storage Agent     → Persists text, summary, verdict, and confidence to SQLite
      ↓
[Result Displayed to User]
```

Early termination is triggered if the OCR agent finds no text in the image (`NO_TEXT_FOUND`).

### 6.3 Data Model

The SQLite database stores one record per unique message:

| Field | Type | Description |
|---|---|---|
| `id` | INTEGER | Auto-incremented primary key |
| `text` | TEXT | Raw extracted text from the image |
| `summary` | TEXT | User-friendly analysis summary |
| `is_scam` | BOOLEAN | 1 = scam, 0 = not scam |
| `confidence_level` | INTEGER | 1 (low) to 5 (high confidence) |
| `created_at` | TIMESTAMP | Record creation time |
| `message_hash` | TEXT | SHA-256 hash for deduplication |

## 7. Functional Requirements

### 7.1 Image Input
- The system must accept images in common formats (JPEG, PNG, etc.)
- The system must pass the image directly to GPT-4o-mini for vision-based OCR
- The system must support pre-loaded example images for first-time users

### 7.2 Text Extraction
- The OCR agent must extract and clean all readable text from the image
- If no text is found, the pipeline must terminate early and notify the user

### 7.3 URL Verification
- The Link Checker must identify all URLs within the extracted text
- URLs must be checked against Google Safe Browsing API v4
- Shortened URLs must be expanded before checking
- Threat categories checked: Malware, Social Engineering, Unwanted Software, Potentially Harmful Application

### 7.4 Content Analysis
- The Content Analyst must identify: urgency indicators, threats, pressure tactics, impersonation, offers that seem too good to be true, and logical inconsistencies
- The analysis must incorporate URL safety results when available

### 7.5 Verdict
- The Decision Maker must return a binary determination: scam or not scam
- The verdict must include a confidence level from 1 (uncertain) to 5 (highly confident)
- The reasoning behind the verdict must be articulated clearly

### 7.6 Summary
- The Summary Agent must produce a single paragraph in plain, non-technical language
- The summary must include actionable recommendations for the user

### 7.7 Language Support
- The system must detect the source language of the extracted text
- If the source language is not English, the summary must be translated into that language
- Currently validated languages: English, Spanish, French, Norwegian

### 7.8 Data Persistence
- Every completed analysis must be saved to the SQLite database
- Duplicate messages (same SHA-256 hash) must not create new records
- All four fields (text, summary, verdict, confidence) are mandatory before storing

### 7.9 Workflow Streaming
- The Gradio interface must stream each agent's output in real time
- Streaming serves as an educational tool showing users how the analysis progresses

## 8. Non-Functional Requirements

| Category | Requirement |
|---|---|
| Latency | Analysis should complete within 30 seconds for typical inputs |
| Concurrency | System must support up to 5 simultaneous analyses |
| Accuracy | Target >85% precision/recall on benchmark scam categories |
| Reliability | Graceful error handling with user-friendly fallback messages |
| Security | API keys stored as environment secrets, never hardcoded |
| Privacy | No image data is retained; only extracted text and analysis results are stored |
| Extensibility | Agent prompts are YAML-configured and can be updated without code changes |

## 9. User Interface

The Gradio-based UI consists of a single tab with:

- **Image upload widget** — drag and drop or click to upload a screenshot
- **Analysis Result textbox** — displays the final plain-language verdict
- **Agentic Workflow panel** — live-streaming HTML log of each agent's reasoning
- **Example gallery** — labeled sample screenshots (EN/ES/FR, various scam types) so users can immediately try the system without having a real scam screenshot

Example categories pre-loaded:
- EN: Gift Scam (Social)
- ES: Banking Scam (Social)
- EN: Billing Scam (SMS)
- EN: Multifactor Authentication Scam (Email)
- EN: Customer Service Scam (Twitter)
- Landscape image with no text (HAM — not a scam)
- FR: Opera Ticket (HAM — not a scam)

## 11. Known Limitations

- **No persistent image storage**: Images are only processed in memory; they cannot be retrieved after analysis
- **OCR accuracy**: Handwritten text, low-resolution images, or heavy visual noise may degrade extraction quality
- **Language coverage**: Translation is best-effort; rare or low-resource languages may be less accurate
- **Pre-release dependencies**: AutoGen v0.4.0.dev11 is a development release and may contain breaking changes
- **Single-image input**: The system currently processes one screenshot at a time, not multi-page documents

## 13. Success Criteria

The product is considered successful when:

1. A non-technical user can upload a screenshot and receive a verdict in under 30 seconds with no configuration required
2. The system correctly identifies >85% of scam messages in the evaluation benchmark
3. Summaries are correctly translated for at least 3 non-English languages
4. All analysis results are reliably archived with no data loss
5. The streaming workflow log is educational and comprehensible to lay users
